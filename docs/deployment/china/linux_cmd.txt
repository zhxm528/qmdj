### Ubuntu 24.04.3 LTS / x86_64 上，用官方 Docker apt 源安装 Docker Engine + Docker Compose 插件
- 卸载可能冲突的旧版本（如果之前装过）
```bash
sudo apt remove docker.io docker-doc docker-compose docker-compose-v2 podman-docker containerd runc
```
- 安装基础依赖
```bash
sudo apt update
sudo apt install -y ca-certificates curl gnupg
```
- 添加 Docker 官方 GPG key
```bash
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
  | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
```
- 添加 Docker 官方 apt 软件源
```bash
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo $UBUNTU_CODENAME) stable" \
  | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```
- 安装 Docker Engine + CLI + containerd
```bash
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin
```
```bash
sudo systemctl status docker
```
- 安装 Docker Compose 插件
```bash
sudo apt install -y docker-compose-plugin
```
```bash
docker compose version
```

- 查看 Docker 版本：
```bash
docker version
```

- 查看服务是否开机自启（一般默认启用）：
```bash
sudo systemctl enable docker
```


- 拉取官方 PostgreSQL 镜像

- 先在宿主机创建数据目录（可选但推荐）：

### 用 docker-compose 管理 PostgreSQL
- 新建一个目录
```bash
mkdir -p ~/pg-stack
cd ~/pg-stack
```

- 新建 docker-compose.yml
```bash
nano docker-compose.yml
```


- 内容示例：
```bash
version: '3.9'

services:
  postgres:
    image: postgres:16
    container_name: qmdj
    restart: unless-stopped
    environment:
      POSTGRES_DB: qmdj
      POSTGRES_USER: mz
      POSTGRES_PASSWORD: 12qw!@QW
    ports:
      - "5432:5432"
    volumes:
      - ./data:/var/lib/postgresql/data
```


保存后（Ctrl+O 回车，Ctrl+X 退出）。



- 启动服务
```bash
docker compose up -d
```
- 查看状态：
```bash
docker compose ps
```
- 停止服务：
```bash
docker compose down
```


- 查看日志：
```bash
docker logs -f qmdj
```
- 重启容器：
```bash
docker restart qmdj
```
- 停止容器：
```bash

```
- 删除容器（不会删映射到宿主机的数据目录）：

### 使用 PM2 守护进程（让服务挂后台）
- 安装 PM2
```bash
npm install -g pm2
```

2. 用 PM2 启动 Next.js 在项目根目录：
```bash
pm2 start npm --name "qmdj" -- run start
```

- 常用命令：
pm2 status           # 查看状态
pm2 logs qmdj    # 看日志
pm2 restart qmdj # 重启
pm2 stop qmdj    # 停止

- 设置开机自启
```bash
pm2 startup systemd
# 终端会输出一条命令，照着再执行一次，例如：
# sudo env PATH=$PATH:/home/youruser/.nvm/versions/node/v20.x/bin pm2 startup systemd -u youruser --hp /home/youruser

pm2 save  
# 保存当前进程列表
```

用 Nginx 做反向代理（绑定域名 / 使用 80 端口）
1. 安装 Nginx
sudo apt install nginx -y


确认 Nginx 正常跑着：

systemctl status nginx

2. 配置站点

新建一个配置文件（举例域名为 example.com）：

sudo nano /etc/nginx/sites-available/next-app


写入：

server {
    listen 80;
    server_name example.com www.example.com;

    location / {
        proxy_pass http://127.0.0.1:3000;  # Next.js 的内部端口

        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;

        # 可选：真实 IP 透传
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}


保存退出后，创建软链接启用配置：

sudo ln -s /etc/nginx/sites-available/next-app /etc/nginx/sites-enabled/next-app

# 检查语法
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx

3. 防火墙放行（如果启用了 ufw）
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable   # 若尚未启用


现在访问：

http://example.com
就会通过 Nginx 转发到 127.0.0.1:3000 的 Next.js 应用。

七、可选：配置 HTTPS（Let’s Encrypt）

如果你有域名，推荐用 Certbot 自动申请证书：

sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d example.com -d www.example.com


之后访问 https://example.com 即可。
