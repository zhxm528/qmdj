# 核心功能：Admin 面板 & 菜单自动生成规则

本文件用于**用自然语言指令驱动 Admin 后台的代码生成和页面结构搭建**。  
通过在本文件中写一句话指令（如“创建某某面板”“在某某面板中，创建某某菜单”），执行本 md 后，将自动：

- 在前台页面 `/app/admin/page.tsx` 中创建对应的**面板**和**菜单入口**
- 在 `app/admin` 目录下生成对应的**面板 / 菜单前台页面目录**及 `page.tsx`
- 在 `app/api/admin` 目录下生成对应的**后台 API 目录**及 `route.ts`
- 在 `md/app/admin` 目录下生成对应的**规则文件目录**及 `cmd.md`

---

## 变量定义

本文件中约定两个核心变量，占位用，实际执行时由“某某”具体名字替换：

- **`{{面板}}`**
  - 含义：一个后台面板（Panel）的名称
  - 示例：`系统管理`、`提示词管理`、`数据分析`
- **`{{菜单}}`**
  - 含义：某个面板下的一个菜单（Menu）名称
  - 示例：`用户管理`、`上下文工程`、`报表中心`

后文所有出现的 `{{面板}}` / `{{菜单}}`，在执行时都会用真实的中文名称替换。

### 路径命名规则

**重要**：在创建前台和后端目录时，需要将面板名称和菜单名称翻译成英文：

- **面板名称翻译**：中文面板名称需要转换为对应的英文标识符
  - 示例：`系统管理` → `system`，`提示词管理` → `prompt`，`会员管理` → `member`
- **菜单名称翻译**：中文菜单名称需要转换为对应的英文标识符
  - 示例：`用户管理` → `users`，`上下文工程` → `context`，`会员卡号` → `card_number`
- **翻译规则**：
  - 优先使用常见英文单词或短语
  - 多个词使用下划线 `_` 连接（如 `card_number`）
  - 保持简洁、语义清晰
  - 在 `/app/admin/page.tsx` 中显示时仍使用中文名称，但路径（path）使用英文翻译

## 指令语法约定

### 1. 创建新面板

- **指令模板**
  - 文本格式：`创建新面板{{面板}}`
- **执行效果**
  - 在前台页面 `/app/admin/page.tsx` 中：
    - 新增一个标题为 **{{面板}}** 的面板区域（显示中文名称）
  - 在代码目录中（使用英文翻译）：
    - 创建前台目录：`/app/admin/{{面板英文}}/`
      - 预留：可在需要时为该面板下的默认页面补充 `page.tsx`
    - 创建后台目录：`/app/api/admin/{{面板英文}}/`
      - 预留：可在需要时为该面板级别的 API 补充 `route.ts`
    - 创建规则目录：`/md/app/admin/{{面板}}/`（规则文件目录可使用中文，便于阅读）

> 说明：创建面板本身不强制生成页面和 API，只搭好面板和目录结构，真正功能通常通过“创建菜单”来落地。

### 2. 在面板下创建菜单

- **指令模板**
  - 文本格式：`在{{面板}}面板中，创建{{菜单}}菜单`
- **执行效果**
  1. **前台页面**
     - 在 `/app/admin/page.tsx` 的 **{{面板}}** 面板中：
       - 新增一个名为 **{{菜单}}** 的菜单入口按钮 / 链接（显示中文名称）
       - 点击该菜单时，在新的浏览器页签打开 `/app/admin/{{面板英文}}/{{菜单英文}}/page.tsx`（路径使用英文翻译）
  2. **前台目录 & 模板页面**（使用英文翻译）
     - 创建目录：`/app/admin/{{面板英文}}/{{菜单英文}}/`
     - 创建文件：`/app/admin/{{面板英文}}/{{菜单英文}}/page.tsx`
       - 内容为一个基础的 React 页面模板，后续可按具体业务修改
  3. **后台 API 目录 & 程序**（使用英文翻译）
     - 创建目录：`/app/api/admin/{{面板英文}}/{{菜单英文}}/`
     - 创建文件：`/app/api/admin/{{面板英文}}/{{菜单英文}}/route.ts`
       - 内容为一个基础的 Next.js Route Handler 模板（例如导出 `GET`/`POST` 占位函数）
  4. **规则文件目录 & 模板**（可使用中文，便于阅读）
     - 创建目录：`/md/app/admin/{{面板}}/{{菜单}}/`
     - 创建文件：`/md/app/admin/{{面板}}/{{菜单}}/cmd.md`
       - 作为该菜单自身的“子规则文件”，用于描述该菜单的具体功能、子操作与自动化规则

> **注意**：`{{面板英文}}` 和 `{{菜单英文}}` 表示将中文名称翻译成对应的英文标识符。例如：
> - `系统管理` → `system`
> - `用户管理` → `users`
> - `会员管理` → `member`
> - `会员卡号` → `card_number`

## 执行规则

- **本 md 的执行范围**
  - 只处理本节下 **「需要执行的操作」** 中的自然语言指令
  - 每一条指令一行，建议使用“- ”开头的列表形式，方便解析
- **执行完成后的整理规则**
  - 执行完本文件后：
    - 会将“**需要执行的操作**”至“**已执行完毕、忽略不执行的操作**”之间的所有内容
    - **剪切**并 **追加到文件末尾**（作为历史操作记录）
  - “需要执行的操作”区块应被清空，准备下次填写新指令
