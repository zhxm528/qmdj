# 受克泄耗（制化）判定说明（证据条目 + 权重）

目标：把“日主强弱/身态”的“受克泄耗”部分从感觉判断，转成可追踪的证据条目与可调权重。

## 总体思路
- 以“证据条目 + 权重”评分模型为主。
- 每一条证据包含：来源、方向（泄/耗/克/制化）、分值、理由。
- 汇总：同类证据叠加，最终形成“受克泄耗总分”与可解释的条目列表。

## 程序调用流程
`/app/bazi/page.tsx` -> `/app/api/bazi/step3.ts` -> `/app/api/bazi/ke_xie.ts`

## 证据条目构成
建议统一结构（示例字段）：
- type: `泄` / `耗` / `克` / `制化`
- source: `食伤` / `财` / `官杀`
- strength: `旺/相/休/囚/死` 或数值
- flags: `透` / `有根` / `成势` / `合化`
- score: 数值分
- reason: 人类可读说明

## 证据来源与计算路径

### 1) 食伤泄身强弱（食伤是否旺、是否透、有根）
证据要点：
- 食伤对应五行是否旺（得令/季节状态）
- 食伤是否透干（天干透出）
- 食伤是否有根（藏干通根）

程序来源：
- 得令/季节状态：`/app/api/bazi/step3.ts` 里的月令/季节状态（或快照表）
- 透干：`/app/api/bazi/tougan/route.ts`
- 通根：`/app/api/bazi/tonggen/route.ts`

建议评分：
- 透干：+w_tougan
- 通根：+w_tonggen
- 旺/相/休/囚/死：按权重表映射成数值（如 1.2/1.0/0.7/0.5/0.3）
- 证据条目示例：`{type:"泄", source:"食伤", flags:["透","有根"], strength:"旺", score:0.85}`

### 2) 财星耗身强弱（财星耗身）
证据要点：
- 财星对应五行强弱（得令/季节状态）
- 财星是否透干
- 财星是否有根

程序来源：
- 得令/季节状态：同上
- 透干：`/app/api/bazi/tougan/route.ts`
- 通根：`/app/api/bazi/tonggen/route.ts`

建议评分：
- 透干 + 通根 + 季节强弱
- 证据条目示例：`{type:"耗", source:"财", flags:["透"], strength:"相", score:0.55}`

### 3) 官杀克身强弱（官杀是否旺、是否成势）
证据要点：
- 官杀对应五行强弱（得令/季节状态）
- 官杀是否透干
- 官杀是否有根
- 官杀是否成势（多柱同时出现、合成局/半合势）

程序来源：
- 得令/季节状态：同上
- 透干：`/app/api/bazi/tougan/route.ts`
- 通根：`/app/api/bazi/tonggen/route.ts`
- 成势判定：可从四柱分布统计 + 合局判断（`/app/api/bazi/ganhe/route.ts` 或合局规则）

建议评分：
- 成势加权：+w_chengshi
- 证据条目示例：`{type:"克", source:"官杀", flags:["有根","成势"], strength:"旺", score:1.05}`

### 4) 制化（以化解克泄耗）
证据要点：
- 有无合化/制化条件（天干五合、地支三合/三会/六合）
- 制化是否成势（合化后五行是否得令/有根）

程序来源：
- 合化规则：`/app/api/bazi/ganhe/route.ts` 或合化规则表
- 结合季节状态评估“化神”强弱

建议评分：
- 以抵消项形式加入：`score` 为负值（抵消克泄耗）
- 证据条目示例：`{type:"制化", source:"合化", flags:["成"], strength:"相", score:-0.40}`

## 汇总建议（示例）
- 受克泄耗总分 = 食伤泄分 + 财耗分 + 官杀克分 + 制化抵消分
- 规则建议先硬编码默认权重，后续放入 `dict_*_ruleset` 表以便可调

## 输出建议（用于“旺衰：日主强弱与身态”）
- 输出：总分 + 分项 + 证据条目列表
- 用于判定：结合得令/通根/得助，形成“强/弱/偏强/偏弱/谨慎判从”的解释链
