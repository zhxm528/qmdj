 - 构建 DeepSeek API 调用参数时，需要以 role: "system" 形式，从以下几个方面考虑如何构建如下提示词：
 - 角色设定
 - 盘已经起好，不用再起盘
 - 给出盘面数据（JSON）
 - 谁（当事人信息）
 - 问什么（事类 + 具体问题）
 - 时间范围
 - 想要的回答形式
 - 背景与动机（案情简述）
 - 流派与规则约束（用什么体系、不引入什么）
 - 术语与解释要求（专业 + 白话）
 - 不确定性与风险提示（允许说“我不太确定”的部分）
 - 回避内容 / 边界（不要什么：鸡汤、医疗建议、极端措辞等）
 - 互动方式（可不可以反问，还是直接断）
 - 输出结构模板（统一的标题/段落格式）
 - 学习/结果侧重（先结论 or 先教学）
